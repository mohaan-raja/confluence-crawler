Title: Form Service


Form service provides the JSON to capture for constructing the form for LEAD, CONTACT, DEAL, ACCOUNT

This is the one of main endpoint which needs the freshsales app to run

The JSON consists of key  field which consists  array of nested fields

Types of fields

  • section
  • checkbox
  • radio
  • dropdown
  • text
  • group_field

Multiple props associated with each fields

Targetable Type, Targetable Id These two fields will be responsible for when lead is converted this respective field value is mapped to Entity(Contact) and field.

regex - This is used when validation for a field validation

fields - The children fields

visible - Whether to be visible in Form

field_options - Custom Hash for application specific flags

parent_id - The parent id of field


 Field Serializer 

This is first step where we modify fields and sets up the relationship between the entity's 


Field Model

_fieldMeta is hash to indicate what template partial to be rendered , and lookup path (the value path ex: model.name, model.leadStage.name)


Ember Dynamic Form


Addon Dynamic form component where we give schema and model


Components

modals/Entity form  will act as Base component for all entity form(lead from, contact form etc)

There are two UI forms Quick view form and Full entity form.
