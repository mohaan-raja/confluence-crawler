Retrospective


What did we do well?

  • PM Stands were faster compared to last sprint. (Dashboard Changes).
  • Code Reviews were faster (Dashboard Changes).
  • Able to Deliver on Committed Tasks.
  • Easier to scale up to the Agile Process and On-boarded Properly.
  • Faster Pace in finishing Tickets and Moving in the proper direction.

What should we have done better?

  • Frontend Code Reviews took longer.
  • Too many Iteration of Code reviews in Frontend.
  • Deployment took much longer than expected ( Items stayed in Staging for two days before deployment was done.)
  • Changes suggested last minute from the PM side.
  • Too many Ad-hoc tasks that were not planned in sprint effort day calculations ( discussions, meetings, etc that does not have proper way of being recorded )

Actions

  • Surface blockers and Other delays to the Team and EMs as soon as Possible.
  • Record all decisions made with the PM in the ticket.
  • Need to an extra day each sprint to consider ad-hoc Items.
  • Need a way to handle definition of done for tickets on Sprint, so that they can be considered as Done when the sprint ends and not be un-delivered Items.
