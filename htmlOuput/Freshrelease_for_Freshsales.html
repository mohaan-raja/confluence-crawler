Test Cases


Test Cases

  • The fields - Title, Severity, Section, Test case status, Automation status, and Owner are mandatory while adding a new test case.
  • Use the below sample csv for importing new test cases into Freshrelease

TC_upload_freshrelease.c…

  • Make sure the mandatory fields are filled in the csv, before importing the cases
  • The "Linked work item" column to contain the work item "FSALES-...." for which the test cases are written
  • For test case updates, enter the corresponding test case ID in the first column, and select "Match records by ID" while importing the csv


Test Runs

  • Test run to be created for every feature testing cycle, and to be assigned to the corresponding QA
  • Test cases are added to the test run, by applying the "linked work item" filter
  • Select test cases within the test run, and add results for the same
  • For 'Failed' test cases, link the corresponding iteration bugs
  • Overall run progress can be tracked inside the corresponding work item
