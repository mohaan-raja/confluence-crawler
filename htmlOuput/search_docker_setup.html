Title: search docker setup

Install docker in your local machine if you don't have already

  1. Download docker dmg from here and install - https://store.docker.com/editions/community/docker-ce-desktop-mac
  2. When you are done, try it by executing the following commands- 
    1. docker run hello-world
    2. If you get a greeting message like "Hello from Docker!", Bingo. :) 
  3. Pull this search service repo in your local directory - search_service_repo, branch - SEARCH-689
    1. git clone git@github.com:freshdesk/search-service.git
    2. cd search-service
    3. Point to the latest staging branch
  4.  Once you clone the repo, you need to place a jar inside the search_service directory which can be downloaded from this link
  5. Follow the steps mentioned in the confluence page from the search service for running the docker. or execute the below commands - 
    1. cd path/to/search-service
    2. docker-compose build --no-cache --build-arg NAME=$LOGNAME --build-arg PASS=<YOUR_SYSTEM_PASSWORD>
    3. docker-compose up -d
  6. Once the docker is up the service will be exposed to port 8080 and ES can be accessed via port 9103. you can use the search service, if you want to stop or start the service anytime you can use the below commands.
    1. docker-compose start
    2. docker-compose stop
  7. If there are any issues in setting up the search service, run the command: docker-compose down -v. Then, retry the initial setup after increasing the memory limit to 4GB in Preferences → Resources in docker desktop.

Troubleshooting:


Docker compose command not found
download docker compose via brew install docker-compose

Error:  product "freshsales" not found

Soln: Open Docker management console and increase memory, clear all images and try from step 5.


Error: Account [1232323] not found

Soln: execute "Search::Strategies::SearchService::Client.new(account_id).tenant_bootstrap" in IRB console
