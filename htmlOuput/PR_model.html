Title: PR model

As part of our shift towards continuous deployment, it would be right time to adopt Pull Requests(PR) to collaborate during code review and eventually ship the code. Below are the proposed changes,

  • We will discontinue the use of wall for code review as we would do code review using GitHub
  • Please raise 'PR for all features and enhancements from your development branch to staging branch
  • Please tag the reviewers like your squad lead. We will also introduce CODEOWNERS later.
  • Once reviewed, PR will be merged to staging branch by the reviewer
  • Please raise PR for bugs against staging branch from your development branch. We will discontinue the usage of sprint branch as the usage has been low. We will monitor the bug flow to understand the impact of this change
  • For larger epics, a branch(& PR) has to be created whenever one or more functionalities are to be reviewed and shipped in phases
  • We will enable protected branch in GitHub so any code merge to staging will require at least 1 approval for the PR


Reference for PR : https://guides.github.com/introduction/flow/ & https://gist.github.com/vlandham/3b2b79c40bc7353ae95
