Title: How to use Javascript Library in conjunction with Server side Library?

You might be interested in tracking user’s activities on your website and their behaviour inside your application after signup. 

Know how to track your user’s lifecycle using client side Javascript library along with our server side libraries with this doc. 

This document explains about how you can use our client side Javascript library in conjunction with one of our server side libraries so that you can track the complete lifecycle of a user.

Step1:

Setup the Javascript library in your Website as mentioned in this documentation. Once you have done this, Freshsales will start tracking your website users ( i.e, what are the pages your users are viewing) by assigning an anonymous ID to each of your visitors.

You can also create events as mentioned in Track Events section mentioned in the above document.

CAUTION:

Since you are interested in creation the Leads from server side, you can turning off the auto-form-capturing feature. In order to do that, you should replace true with false at the end of the above installation script.

                       

Step2:

Based on the language your Application is built on, choose any of our server side libraries and set it up in your application. Refer this for more details on available libraries. You can follow the instructions specified under “Getting Started” for setting it up.

If you want to use Java Library, we would recommend you to download the latest version of Java library instead of using the version specified in document.

Step3:

When an user Submits the form on your website, apart from the form data you should also pass the anonymous ID of the user set by Freshsales to your Servers.

You can get the anonymous ID on client side as show below:

  

freshsales.anonymous_id



Step4:

Once you are done storing the signup data on your server, you need to make a network call to Freshsales in order to alias anonymous ID to the User ID of your application.

User ID is the unique identification that you can give to your user. Check  UserID spec to know more about UserID.

Aliasing can be done as shown below:


FreshsalesAnalytics freshsales = new FreshsalesAnalytics("FRESHSALESURL”,"FRESHSALESAPP_TOKEN");

freshsales.alias("anonymousidentifier", "uniqueidentifier”);



Step5:

Once you are done aliasing the anonymous ID to User ID, you can make a identify call to Freshsales passing the user data, which results in creating a Lead.

You can even use the other methods of the library such as “trackEvent” and “trackPageView” as per need. You can refer this  documentation for more details on these methods usage.


CAUTION:

You need to make sure that you call the alias method before identify in order for this setup to work as expected.

Title: How to use Javascript Library in conjunction with Freshsales API ?

You might be interested in tracking user’s activities on your website but create the Lead from your Server side using Freshsales's Lead creation API upon your Website Form submission. But  that question is how to link the visitor activities on the Website to the Lead which you have created? - This document will give the walkthrough on how to solve this use-case.


Step1:

Setup the Javascript library in your Website as mentioned in this documentation. Once you have done this, Freshsales will start tracking your website users ( i.e, what are the pages your users are viewing) by assigning an anonymous ID to each of your visitors.

You can also create events as mentioned in Track Events section mentioned in the above document.

CAUTION:

Since you are interested in creation the Leads from server side, you can turning off the auto-form-capturing feature. In order to do that, you should replace true with false at the end of the above installation script.

Step2:

When an user Submits the form on your website, apart from the form data you should also pass the anonymous ID of the user set by Freshsales to your Servers.

You can get the anonymous ID on client side as show below:

  

freshsales.anonymous_id


Step3:

Once you are done storing the form data on your servers, you can make a API call to Freshsales to create a Lead using your Form data. You can go through this documentation to know more about Freshsales APIs.

Once the above Lead creation is successful, you will get the Freshsales's Lead ID in the response. Using this Lead ID and  anonymous ID, you should be making the below API call which will link the visitor activities on the Website to the Lead.

Type: POST

URL : /api/leads/{{LEAD-ID}}/associate_to_visitor

BODY: {

  cookie: anonymous-ID, 

}
